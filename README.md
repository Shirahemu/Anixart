# Anixart iOS Port

Версия: `0.1`

Anixart iOS Port - это перенос Android-приложения Anixart на iPhone и iPad. Цель проекта - сохранить привычный сценарий просмотра аниме: авторизация, главная лента, поиск, карточки релизов, профиль, списки и подготовка к воспроизведению эпизодов, но собрать все это как нативное SwiftUI-приложение для iOS.

Проект находится на раннем этапе разработки. Сейчас упор сделан на сетевой слой, совместимость с API Anixart, безопасную диагностику и базовый пользовательский интерфейс.

## Что уже есть

- SwiftUI-приложение с вкладками: Главная, Поиск, Списки, Профиль и Настройки.
- Авторизация через Anixart API с сохранением токена в Keychain.
- Mock mode для разработки интерфейса без сетевых запросов.
- Live mode с выбором основного, альтернативного или пользовательского API URL.
- Профили заголовков: прозрачный iOS, Android compatibility, Exact Android 8.5.2.
- Генератор `Sign`, совместимый с Android-клиентом.
- Главная лента на основе расписания и фильтров релизов.
- Поиск релизов.
- Экран релиза с постером, описанием, жанрами, метриками, похожими релизами, рекомендациями и списком эпизодов.
- Экран профиля со статистикой, списками, историей, голосами, комментариями и друзьями.
- Developer Tools для проверки логина, профиля, релиза, endpoint-тестов, настроек и диагностических событий.
- Редакция чувствительных данных в логах: token, password, Sign, Authorization, Cookie и Set-Cookie.
- Unit-тесты для endpoint-метаданных, декодирования ответов, профилей, релизов, настроек, диагностики и генерации Sign.

## Текущее состояние

На версии `0.1` приложение можно использовать как основу iOS-порта и как диагностический клиент для проверки API-совместимости. Воспроизведение видео и пользовательские действия вроде регистрации, голосования, избранного, комментариев и жалоб пока не включены в основной пользовательский поток.

По умолчанию включен Mock mode, чтобы приложение запускалось без аккаунта и без сети. Для проверки live API откройте Настройки и отключите Mock mode.

## Структура проекта

```text
Anixart/
  App/                  Состояние приложения, настройки, root navigation
  Core/
    Diagnostics/        События диагностики, аудит декодирования, редактирование логов
    Models/             DTO и модели ответов Anixart API

    Storage/            Keychain и in-memory хранилища токена
    Utilities/          Форматирование JSON, безопасное логирование, redaction
  Features/             SwiftUI-экраны приложения
  Services/             Сервисный слой поверх API endpoints
AnixartTests/           Unit-тесты
DEVELOPMENT.md          Ручные regression checks и планы проверки
```

## Запуск

1. Откройте `Anixart.xcodeproj` в Xcode.
2. Выберите схему `Anixart`.
3. Запустите приложение на iOS Simulator или устройстве.
4. Для разработки без сети оставьте `Mock mode` включенным.
5. Для live API откройте Настройки, отключите `Mock mode`, выберите сервер и профиль заголовков.

## Настройки live mode

В приложении доступны:

- `Primary` API: `https://api-s.anixsekai.com/`
- `Alternate` API: `https://api-s2.anixart.tv/`
- `Custom` API URL
- профили User-Agent и совместимости с Android-клиентом
- включение или отключение заголовка `Sign`
- verbose diagnostics

Если live API меняет формат ответа, откройте Settings -> Developer Tools -> Diagnostics. Там можно увидеть redacted-события сети и декодирования без утечки токена или пароля.

## Тесты

Тесты запускаются из Xcode через схему `Anixart` или командой:

```sh
xcodebuild test -scheme Anixart -destination 'platform=iOS Simulator,name=iPhone 16'
```

Если в локальном Xcode нет указанного симулятора, выберите доступное устройство в списке симуляторов.

## Безопасность и ограничения

Проект не хранит пароль пользователя. Токен live-сессии сохраняется через Keychain, а mock-сессия живет только в памяти. Диагностика и debug output проходят через redaction policy, чтобы не выводить секреты в UI и логи.

Это не официальный клиент Anixart. Проект предназначен для портирования опыта Android-приложения на iOS и исследования совместимости API.
