Версия: `0.4`

Anixart iOS Port - это нативный SwiftUI-порт Android-приложения Anixart для iPhone и iPad. Цель проекта - перенести привычный сценарий просмотра аниме на iOS: авторизация, главная лента, поиск, карточка тайтла, списки профиля, избранное, профиль, диагностика API и воспроизведение эпизодов.

Проект не является официальным клиентом Anixart. Он развивается как исследовательский iOS-порт с упором на совместимость с существующим API, безопасную диагностику и аккуратный пользовательский интерфейс.

## Что есть в версии 0.4

- SwiftUI-приложение с нижними вкладками: Главная, Списки, Профиль и Настройки.
- Поиск доступен из шапки Главной, без отдельной вкладки в tab bar.
- Live mode для работы с API Anixart и Mock mode для разработки без сети.
- Авторизация через Anixart API с хранением live-токена в Keychain.
- Профили заголовков и Android-compatible Sign для проверки совместимости с Android-клиентом.
- Главная лента на `filter/0` с вкладкой `Последнее`, сортировкой по `episode_last_update` и карточками в формате `12 из 12 эпизодов • 4.7 ★`.
- Экран тайтла с production UI: hero-блок, постер, возраст/оценка/комментарии, описание, информация, жанры, скриншоты, связанные тайтлы, рекомендации, рейтинг и комментарии.
- Избранное и статус списка профиля синхронизируются с сервером через `favorite/add|delete` и `profile/list/add|delete`.
- Watch-карта объединяет выбор типа, источника и эпизода, затем открывает плеер.
- Player v1 поддерживает прямые видео-ссылки через AVPlayer и WebView fallback для iframe/Web-плееров, включая AniLibria/Liberty-сценарии.
- Комментарии релиза: список, ответы, добавление, редактирование, удаление, голосование и жалобы через Android-compatible endpoints.
- Рейтинг тайтла: средняя оценка, количество голосов, распределение 5..1, текущая оценка пользователя, установка, изменение и удаление оценки.
- Профиль с аватаром, статистикой, списками, рейтингами, друзьями, динамикой просмотров и превью истории.
- Полная история просмотров из профиля: пагинация, строки в Android-стиле, удаление записей и фоновая запись просмотра при открытии плеера.
- Списки профиля с вкладками, корректным mapping статусов и автозагрузкой.
- Полный redacted diagnostics export: события сети, UI, player trace и профильный audit без токенов, паролей и cookie.
- Unit-тесты для endpoint-метаданных, декодирования релизов/эпизодов/профиля/истории/рейтинга, diagnostics redaction, Sign и player URL handling.

## Что изменилось в 0.4

- Добавлены полноценные комментарии релиза с совместимыми телами `release/comment/add`, `release/comment/edit`, `report/comment/release` и обработкой типовых ошибок API.
- Добавлен пользовательский рейтинг тайтла через `release/vote/add/{r_id}/{vote}` и `release/vote/delete/{r_id}` с откатом UI при ошибке.
- Добавлена история просмотров: превью в профиле, отдельный экран `История просмотров`, загрузка `history/{page}`, удаление `history/delete/{r_id}` и запись `history/add/{r_id}/{s_id}/{position}` при открытии плеера.
- Расширены mock-ответы, diagnostics и endpoint/model tests для новых сценариев.
- Очищены персональные Xcode signing-настройки из проекта.

## Текущее состояние

Версия `0.4` уже пригодна как рабочий прототип iOS-порта и диагностический клиент для проверки API-совместимости. Основные пользовательские маршруты собраны: Главная -> тайтл -> Watch -> Player, а также Профиль, Списки, Комментарии, Рейтинг и История просмотров.

Kodik playback остаётся нестабильной внешней WebView-зависимостью и не считается полностью закрытым направлением. AniLibria/Liberty fallback сохранён.

## Запуск

1. Откройте `Anixart.xcodeproj` в Xcode.
2. Выберите схему `Anixart`.
3. Запустите приложение на iOS Simulator или устройстве.
4. Для разработки без сети оставьте `Mock mode` включенным.
5. Для live API откройте Настройки, отключите `Mock mode`, выберите сервер и профиль заголовков.

## Live mode и диагностика

В приложении доступны:

- `Primary` API: `https://api-s.anixsekai.com/`
- `Alternate` API: `https://api-s2.anixart.tv/`
- пользовательский API URL;
- профили заголовков и User-Agent;
- включение или отключение `Sign`;
- verbose diagnostics и Full Trace export.

Диагностика редактирует чувствительные данные перед показом и экспортом: токены, пароли, Authorization, Cookie, Set-Cookie, Sign и подписанные query values.

## Безопасность

Проект не хранит пароль пользователя. Live-токен сохраняется через Keychain, mock-сессия живёт только в памяти. Локальные диагностические экспорты, `.jsonl`, `.log`, `.trace`, `.env` и локальные конфиги исключены из git.

Перед публикацией проверяйте, что в commit не попали реальные пользовательские diagnostics-файлы или персональные Xcode state-файлы.

## Тесты

Тесты запускаются из Xcode через схему `Anixart` или командой:

```sh
xcodebuild test -scheme Anixart -destination 'platform=iOS Simulator,name=iPhone 16'
```

Если в локальном Xcode нет указанного симулятора, выберите доступное устройство в списке симуляторов.
